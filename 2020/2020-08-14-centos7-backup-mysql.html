MySQL 数据库自动备份脚本

kun 2020-08-14 19:30:00 MySQLLinux

定时备份数据库有时是能救命的，别问我是怎么知道的！ 使用 linux 的 crontab 命令和 mysql 的 mysqldump 命令可以很方便的实现对 mysql 数据库的定时备份功能。

# crontab

linux 的 crontab 命令是用来执行周期性定时任务的。系统会每分钟检查一次待执行的任务，有时间匹配的就马上执行。

# 参数

-u

指定用户，不加 -u 的话默认是当前用户。

-l

查询任务列表，如查询 yyk 用户的定时任务：crontab -u yyk -l

-e

打看编辑器，向用户的执行列表添加任务，如向当前用户的执行列表增加一个每天 18 点执行指定脚本的任务

* crontab -e 
* 然后在打开的文件编辑器中输入：00 18 * * * /web-service-jar/goods-service/mysql-backup/test.sh

-r

删除定时用户列表

注意：不使用 -u 指定用户的话默认操作的都是当前用户的定时任务列表

# cron 表达式

cron 表达式是用来指定执行时间的，格式如下

f1 f2 f3 f4 f5 program

其中 f1 是表示分钟，f2 表示小时，f3 表示一个月份中的第几日，f4 表示月份，f5 表示一个星期中的第几天。program 表示要执行的程序。

当 f1 为 * 时表示每分钟都要执行 program，f2 为 * 时表示每小时都要执行程序，其馀类推

当 f1 为 a-b 时表示从第 a 分钟到第 b 分钟这段时间内要执行，f2 为 a-b 时表示从第 a 到第 b 小时都要执行，其馀类推

当 f1 为 */n 时表示每 n 分钟个时间间隔执行一次，f2 为 */n 表示每 n 小时个时间间隔执行一次，其馀类推

当 f1 为 a, b, c,... 时表示第 a, b, c,... 分钟要执行，f2 为 a, b, c,... 时表示第 a, b, c...个小时要执行，其馀类推


*    *    *    *    *
-    -    -    -    -
|    |    |    |    |
|    |    |    |    +----- 星期中星期几 (0 - 7) (星期天 为0)
|    |    |    +---------- 月份 (1 - 12) 
|    |    +--------------- 一个月中的第几天 (1 - 31)
|    +-------------------- 小时 (0 - 23)
+------------------------- 分钟 (0 - 59)

# mysqldump

mysqldump 是常用的 mysql 数据库导出命令，功能参数很多，不一一介绍。举两个例子说明下用法：

备份所有数据库到指定sql文件
mysqldump -uroot -proot --all-databases >/tmp/all.sql

备份指定数据库到指定sql文件
mysqldump -uroot -proot --databases db1 db2 >/tmp/user.sql

# 创建一个备份 mysql 数据库的定时任务

# 1. 编写备份脚本

示例脚本：

#!/bin/bash
DATE=`date +%Y%m%d%H%M` #every minute
DATABASE=springboot-admin #database name
DB_USERNAME=root #database username
DB_PASSWORD="mysql" #database password 
BACKUP_PATH=/backup/mysqldata #backup path

#backup command 
/usr/bin/mysqldump -u$DB_USERNAME -p$DB_PASSWORD -h 127.0.0.1 -R --opt $DATABASE | gzip > ${BACKUP_PATH}\/${DATABASE}_${DATE}.sql.gz #just backup the latest 5 days
find ${BACKUP_PATH} -mtime +5 -name "${DATABASE}_*.sql.gz" -exec rm -f {} \;

# 2. 给脚本授权

chmod +x database_backup_shell.sh

# 3. 创建定时任务

输入命令：
crontab -e

在打开的编辑器中输入如下内容后保存退出：
00 3 * * * /root/database_backup_shell.sh

ok，至此，一个可实现每天凌晨三点自动执行第一步编写的脚本， 备份 mysql 数据库到 /backup/mysqldata 目录， 且只保留最近5天的数据库备份的定时任务就配置完成了！
