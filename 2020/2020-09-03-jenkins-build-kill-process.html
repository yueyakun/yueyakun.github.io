记一次在 jenkins 构建中启动 Jar 包失败的问题

kun 2020-09-03 20:30:00 jenkins

# 问题

最近在尝试使用 jenkins 启动 Jar 包，构建顺利完成，但是后端服务却并没有启动。

构建日志如下：

然而服务并没有启动：

boot.sh 启动脚本的主要命令如下：

nohup java -Xms300m -Xmx300m -jar ${APP_HOME}/${APP_NAME} > /dev/null 2>&1 &

经过一番面向搜索引擎编程之后，找到了原因和解决办法。

# 原因

原来 jenkins 有一个 ProcessTreeKiller 组件，这个组件在构建完成之后会杀死跟这次构建相关的的所有衍生进程。 判断是不是衍生进程的依据就是：该进程启动时的环境变量中的 BUILD_ID。

BUILD_ID 是 jenkins 构建时生成的一个任务号。随着构建次数从 1 开始，依次递增。

# 解决方案

解决方案就是在启动 Jar 包这一行脚本之前修改环境变量 BUILD_ID 的值，启动脚本执行之后再改回来。 其实如果构建任务只是单纯启动 Jar 包的话不改回来也没关系。修改后的构建脚本如下：

#暂存BUILD_ID
OLD_BUILD_ID = $BUILD_ID
#修改BUILD_ID
export BUILD_ID=dontKillMe
#启动 Jar 包
/web-service-jar/house-service/boot.sh house-service.jar restart
#将BUILD_ID改回原来的值
BUILD_ID=$OLD_BUILD_ID
#打印
echo $BUILD_ID

到服务器上查询服务启动结果：

查询该进程的启动环境变量，发现 BUILD_ID 果然已经被修改成了 dontKillMe。查询命令如下：

ps eww -p 14639

ok，至此问题完美解决！
