深入理解 Java 8 的 Stream 流式编程（一）

kun 2020-08-20 20:30:00 StreamJava 8

关于 Java 8 的 Stream 流式编程内容较多，所以分为三部分：

  1. 常用的 Stream API：对于新东西我一般先学会用，然后再慢慢的深入了解。
  2. 关于 Straem 的一些总结：熟练使用之后就该总结一下了。
  3. Stream 的原理：知道工具的原理才能更好的使用工具。

这篇是该日志的第一部分，也就是常用 API 的罗列和用法示例。

# Stream 常用 API

# filter

  • 签名：Stream filter(Predicate<? super T> predicate)

  • 作用：返回一个只包含满足predicate条件元素的Stream。

  • 示例

//获取偶数
Integer[] sixNums = {1, 2, 3, 4, 5, 6};
Integer[] evens =Stream.of(sixNums).filter(n -> n%2 == 0).toArray(Integer[]::new);

# map

  • 签名： Stream map(Function<? super T, ? extends R> mapper)

  • 作用：返回一个对当前所有元素执行执行 mapper 之后的结果组成的 Stream。是一种转换操作，类似的还有 mapToInt、mapToLong 和 mapToDouble。

  • 示例

//求平方
List<Integer> nums = Arrays.asList(1, 2, 3, 4);
List<Integer> squareNums = nums.stream().map(n -> n * n).collect(Collectors.toList());

# flatMap

  • 签名： Stream flatMap(Function<? super T, ? extends Stream<? extends R>> mapper)

  • 作用：对每个元素执行 mapper 指定的操作，并用所有 mapper 返回的 Stream 中的元素组成一个新的 Stream 作为最终返回结果。就是把二维的数据源转换为一维。类似分还有 flatMapToInt、flatMapToLong 和 flatMapToDouble。

  • 示例

//从多个对象获取一个整体的stream
Stream<List<Integer>> inputStream = Stream.of(
    Arrays.asList(1),
    Arrays.asList(2, 3),
    Arrays.asList(4, 5, 6)
);
Stream<Integer> outputStream = inputStream.flatMap((childList) -> childList.stream());

# distinct

  • 签名：Stream distinct()

  • 作用：返回一个去除重复元素之后的Stream。

  • 示例

//去掉重复的单词
Stream<String> stream= Stream.of("I", "love", "you", "too", "too");
stream.distinct().forEach(str -> System.out.println(str));

# sorted

  • 签名：Stream sorted()

  • 作用：按自然顺序排序。

  • 示例

List<Integer> nums = Arrays.asList(3, 2, 1, 4);
nums.stream().sorted().forEach(num-> System.out.println(num));

  • 签名：Stream sorted(Comparator<? super T> comparator)

  • 作用：使用自定义比较器 comparator 排序。

  • 示例

//按长度升序
List<String> strs = Arrays.asList("aa", "a", "aaaa", "aaa");
strs.stream().sorted((a,b)->a.length()-b.length()).forEach(str-> System.out.println(str));

# peek

  • 签名：Stream peek(Consumer<? super T> action)

  • 作用：遍历，与 forEach 不同的是 forEach 是结束操作，而 peek 是中间操作。

  • 示例

List<String> strs = Arrays.asList("aa", "a", "aaaa","aaa");
strs.stream().peek(System.out::println).forEach(System.out::println);

# limit

  • 签名：Stream limit(long maxSize)

  • 作用：截断流，从 Stream 中截取 maxSize 长度的元素。

  • 示例

//打印前两个
List<String> strs = Arrays.asList("aa", "a", "aaaa","aaa");
strs.stream().limit(2).forEach(System.out::println);

# skip

  • 签名：Stream skip(long n)

  • 作用：忽略指定个数的元素。

  • 示例

//跳过前两个
List<String> strs = Arrays.asList("aa", "a", "aaaa","aaa");
strs.stream().skip(2).forEach(System.out::println);

# 其他简单但是实用的 API

Stream 还有其他简单但是十分好用的 API，比如min，max，sum，count，anyMatch，allMatch，noneMatch，findFirst，findAny等，在这里就不一一列举了。

下面着重介绍 reduce 和 cellect 这两个 API。

# reduce

reduce操作可以实现从一组元素中生成一个值，sum()、max()、min()、count()等都是reduce操作，将他们单独设为函数只是因为常用。reduce 方法定义有三种重载形式。

* Optional<T> reduce(BinaryOperator<T> accumulator)
* T reduce(T identity, BinaryOperator<T> accumulator)
* <U> U reduce(U identity, BiFunction<U, ? super T, U> accumulator,  BinaryOperator<U> combiner)

虽然入参越来越多，但是语义并没有变，都是先将两个元素交个 accumulator 处理，然后处理结果在和第三个元素一起交给 accumulator 处理，以此类推，直到所有元素都处理完，就将最后一次 accumulator 函数的处理结果返回。不同的是前两个方法输入和输出的类型一致，第三个方法的输人和输出的类型可以不一致，且第三个方法需要定义一个拼接器，拼接器的作用是指示出并行时，各个部分的处理结果要怎样合并。举个例子：

List<String> words = Arrays.asList("nice", "to", "meet", "you");
//拼接单词
Optional<String> reduce1 = words.stream().reduce((a, b) -> a + " " + b);
System.out.println(reduce1.get());
//加个开头单词
String reduce2 = words.stream().reduce("Hi,",(a, b) -> a + " " + b);
System.out.println(reduce2);
//计算各个单词长度之和
Integer reduce3 = words.parallelStream().reduce(0, (sum, str) ->sum+str.length(), 
(s1, s2) -> s1 + s2);
System.out.println(reduce3);

# collect

这个方法有点复杂，字面意思是收集。属于结束操作，用于收集流中的数据填充指定的容器。方法签名有两个：

* <R> R collect(Supplier<R> supplier, BiConsumer<R, ? super T> accumulator, BiConsumer<R, R> combiner);
* <R, A> R collect(Collector<? super T, A, R> collector)；

两个方法的语义也是一样的，第二个方法的入参 Collector（收集器） 就是第一个方法三个入参的抽象。

收集器的三个入参作用分别是：

* 创建目标容器
* 将元素映射进容器中
* 合并并发情况下多部分的结果

举两个例子说明一下第一个方法的用法：

List<String> words = Arrays.asList("I", "LOVE", "YOU");
//转list
ArrayList<Object> list = words.stream().collect(ArrayList::new, ArrayList::add, ArrayList::addAll);

//转map
HashMap<String,Integer> map = words.stream().collect(HashMap::new, (m,s)->m.put(s,s.length()), HashMap::putAll);

# Java 提供的收集器

大部分时候我们是用不到第一个方法的，因为 Java 为我们提供了大部分常用的收集器。这些收集器都由 java.util.stream.Collectors 类提供。

# 生成 Collection 的收集器

* Collectors.toList()
* Collectors.toSet()
* Collectors.toCollection()

toList() 和 toSet() 得到的结果类型是 List 和 Set，如果要的到具体的实现类类型的集合就需要 toCollection，举个例子：

//得到指定容器类型的结果
ArrayList<String> arrayList = stream.collect(Collectors.toCollection(ArrayList::new));
HashSet<String> hashSet = stream.collect(Collectors.toCollection(HashSet::new));

# 生成 Map 的收集器

  1. Collectors.toMap()
  • 作用：将流中的元素按照规则装换成 map
  • 示例：
// 将User列表转换成id为key，User对象为value的Map
Map<Integer, User> userMap =
     users.stream().collect(Collectors.toMap(User::getId,// 生成key
                                     Function.identity()));// 生成value
  1. Collectors.partitioningBy()
  • 作用：将流中的元素按照规则分成两部分
  • 示例：
// 按是否年满18岁将用户分为两组
Map<Integer, User> userMap =
     users.stream().collect(Collectors.partitioningBy(u -> u.getAge >= 18));
  1. Collectors.groupingBy()
  • 作用：将流中的元素按照规则分组并将分组后的元素再按照指定的收集器进行收集
  • 示例：
//将用户按照年龄分组
Map<Integer, List<User>> userMap1 =
     users.stream().collect(Collectors.groupingBy(User::getAge));
     
//将用户按照年龄分组
Map<Integer, List<User>> userMap2 =
     users.stream().collect(Collectors.groupingBy(User::getAge, Collectors.toList()));
     
//将用户按照年龄分组只保留名字
Map<Integer, List<String>> userMap3 =
     users.stream().collect(Collectors.groupingBy(User::getAge, Collectors.mapping(User::getName, Collectors.toList())));

# 总结

前面基本列出了所有常用的 Stream API，其中 reduce 和 collect 用法稍微有些复杂，其余都很简单。通过对这些 API 的学习我们不难发现，Stream 似乎是对 Java 集合框架的一种加强：

它通过自带的 API 和 lambda 表达式降低了我们处理集合时的代码量；

通过并行特性提高了处理大批量集合元素的效率。
