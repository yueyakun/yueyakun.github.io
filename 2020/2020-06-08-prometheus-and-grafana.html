Actuator + Prometheus + Grafana 实现线上服务状态可视化监控

kun 2020-06-08 18:00:00 ActuatorPrometheusGrafana

实时、可视化的监控线上运行的 Spring Boot 应用是保障应用健康运行的关键。Actuator + Prometheus + Grafana 的组合可以十分方便的实现线上应用监控。

# Prometheus 简介

Prometheus 是由 SoundCloud 开源监控告警解决方案，2012 年开始编写代码，2015 年在 github 上开源。2016 年 Prometheus 成为继 k8s 后，第二名 CNCF(Cloud Native Computing Foundation) 成员.

主要功能和架构原理请参考 官方文档 或者 gitbooks 上的 Prometheus 实战 或者 Prometheus book

# Grafana 简介

Grafana 是一套开源的分析监视平台，支持 Graphite, InfluxDB, OpenTSDB, Prometheus, Elasticsearch, CloudWatch 等数据源，其 UI 非常漂亮且高度定制化。

详细信息可以移步 官方网站 或者 Grafana 全面瓦解

# 在应用中引入 Actuator

关于 Actuator 的使用上篇已经讲过，地址：Spring Boot Actuator 使用详解

# 在项目中引入 Prometheus

  1. 在 pom 文件中引入依赖
<dependency>
	<groupId>io.micrometer</groupId>
	<artifactId>micrometer-registry-prometheus</artifactId>
</dependency>
  1. 放开 Actuator 的 /prometheus 端点
management:
 endpoints:
   web:
     exposure:
       include: prometheus
  1. 增加配置类
@Bean
MeterRegistryCustomizer<MeterRegistry> configurer(@Value("${spring.application.name}") String applicationName) {
	return (registry) -> registry.config().commonTags("application", applicationName);
}
  1. 访问 HTTP 端口，测试端点状态

http://localhost:8083/actuator/prometheus

# CentOS7 安装配置 Grafana

具体安装配置步骤见我的这篇博客：CentOS7 安装配置 Grafana

安装成功后访问 http://ip:3000 ，默认用户名和密码都是 admin，首次登录会提示重置密码。

# 设置数据源

登录后点击左侧菜单中的齿轮打开 config 页面，点击页面中的 add data source 按钮，如下图

数据源类型选择 prometheus

输入数据源名称和 url 地址，如果没有 auth 逻辑直接点击下面的 save and test 按钮就 ok 了。

# 导入监视面板

监视面板可以自己创建，也可以选择导入。导入方法如下：

  1. 在 Grafana 的面板市场选择自己需要的面板模板，面板市场可以按名字、数据源类型等条件进行筛选，这里推荐人气最高的 JVM (Micrometer) ,ID 是 4701 。

点击 + 号菜单中的 import 选项

输入面板 ID 点击 load 按钮

修改面板名称，选择数据源，点击 import 按钮就完成面板导入了。

OK !! 接下来就可以愉快的监控 Spring Boot 服务运行状态了！
