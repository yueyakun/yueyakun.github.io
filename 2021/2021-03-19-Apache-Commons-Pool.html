Apache Commons Pool

kun 2021-03-19 Pool

# 前言

# 池化技术概念

把一些能够复用的东西（比如说数据库连接、线程）放到池中，避免重复创建、销毁的开销，从而提高程序性能。

# 池化技术适用场景

  • 创建数量受限，如数据库连接，redis 连接

  • 创建成本高，如线程对象

# Apache Commons Pool

# 简介

Apache Commons Pool 在是很多著名框架池化技术底层实现，比如我们常见的 Redis 客户端 Jedis 和 Lettuce，它们的连接池都是借助 Apache Commons Pool 实现的。

Apache Commons Pool 软件库提供了一个对象池 API 和一些对象池实现。与 V1 版本 相比，V2 版本完全重写了池的实现。除了性能和可伸缩性的改进，V2 版本还提供了更健壮的实例跟踪和池监控功能。下面我们分析一下 V2 的主要 API。

官方文档

# 主要 API

Apache Commons Pool 的主要接口有三个：

  • PooledObject：资源对象放入池子中之前，用它来包装一下，附加为资源对象提供一些创建时间、状态、最后使用时间之类的状态信息。

  • PooledObjectFactory：负责生成池中对象的工厂类；

  • ObjectPoll：也就是池子本身，负责管理池中的对象资源。

# PooledObject：池化对象包装类

public interface PooledObject<T> extends Comparable<PooledObject<T>> {
    T getObject();//返回被包装的对象本身
    long getCreateTime();//创建时间
    long getActiveTimeMillis();//最後一次处于活动状态的时间
    long getIdleTimeMillis();//最后一次处于空闲状态的时间
    //剩余api也都很简单，不再一一列出
}

# PooledObjectFactory：对象工厂

public interface PooledObjectFactory<T> {
    PooledObject<T> makeObject();//创建一个资源对象实例，并把它包装成可以放入资源池中的PooledObject实例
    void destroyObject(PooledObject<T> p);//摧毁池中不再需要的实例
    boolean validateObject(PooledObject<T> p);//校验这个实例是否可以被池安全地返回
    void activateObject(PooledObject<T> p);//激活一个实例要出池子的实例
    void passivateObject(PooledObject<T> p);//钝化一个返回池子的实例
}

# ObjectPoll：对象池接口

public interface ObjectPool<T> extends Closeable {
    T borrowObject();//从这个池中获取一个实例
    void returnObject(T obj);//将一个实例返还到池中，注意这个对象必须是通过borrowObject()获取的。
    void invalidateObject(T obj);//使池中的一个实例失效，注意这个对象必须是通过borrowObject()获取的。
    void addObject();//创建一个对象并放入池中，一般用来预加载池中的对象资源。
    int getNumIdle();//获取池中空闲资源对象的数量。
    int getNumActive();//获取池中忙碌资源对象的数量。
    void clear();//清除池子中的闲置资源。
    void close();//关闭池子，释放所有资源对象
}

# 默认实现

Apache Commons Pool 提供了一套默认的实现。依靠这套默认实现可以快速构建自己的对象池。

# DefaultPooledObject

默认的池化对象包装类，提供了 PooledObject 接口的一套实现方法。

# BasePooledObjectFactory

PooledObjectFactory 的虚拟实现类，虚拟方法只有 create() 和 wrap(T obj) 两个方法，其他接口方法都是空实现。这个类主要是实现了 PooledObjectFactory 的 makeObject 方法。

用户使用是只需实现下面两个方法：

  • create()：定义如何创建一个新的池化对象

  • wrap(T obj)：定义如何将一个池化对象转换成池化对象的包装类

该虚拟类的其他方法如有需要就实现，没有需要可以不去实现。

# GenericObjectPool

ObjectPool 的实现类，它还继承了 BaseGenericObjectPool（主要定义了池子的一些配置项）。与合适的 PooledObjectFactory 配合可以为任何对象提供健壮的池化功能。

# 使用和测试

下面我们用一个例子熟悉怎么构建一个对象池，也顺便测一测对象池对我们提升程序性能有多大的帮助。我们就以构建一个 JDBC 连接池为例来学习和测试 Apache Commons Pool。

首先，创建一个简单的 Spring Boot 项目，源码已上传 github。

然后创建一个用来生成 JDBC 连接的 PooledObjectFactory 工厂类。

public class ConnectionFactory extends BasePooledObjectFactory<Connection> {
	private Logger logger = LoggerFactory.getLogger(this.getClass().getName());

	String url = "jdbc:mysql://localhost:3306/demo?characterEncoding=UTF8&useSSL=false&serverTimezone=GMT";
	String username = "root";
	String password = "123456";

	static {
		try {
			Class.forName("com.mysql.cj.jdbc.Driver");
		} catch (Exception ex) {
			throw new RuntimeException(ex + "驱动类加载失败！");
		}
	}

	@Override
	public Connection create() throws Exception {
		logger.info("创建新的链接");
		return DriverManager.getConnection(url, username, password);
	}

	@Override
	public PooledObject<Connection> wrap(Connection connection) {
		return new DefaultPooledObject<>(connection);
	}

	@Override
	public void destroyObject(PooledObject<Connection> p) throws Exception {
		Connection connection = p.getObject();
		if (Objects.nonNull(connection)) {
			try {
				connection.close();
				logger.error("关闭连接");
			} catch (SQLException e) {
				logger.error("关闭连接失败");
			}
		}
	}

	@Override
	public void passivateObject(PooledObject<Connection> p) throws Exception {
		logger.info("连接返回连接池");
	}
}

然后创建一个测试 Controller 类，提供两个测试方法，一个使用池化技术，一个不使用池化技术。

@RestController
@RequestMapping("/test")
public class TestController {

	private Logger logger = LoggerFactory.getLogger(this.getClass().getName());

	private GenericObjectPool<Connection> connectionPool;
	private ConnectionFactory connectionFactory;

	@PostConstruct
	public void init() {
		GenericObjectPoolConfig<Connection> poolConfig = new GenericObjectPoolConfig<>();
		poolConfig.setMaxTotal(10);
		poolConfig.setMaxIdle(10);
		poolConfig.setMinIdle(0);
		poolConfig.setLifo(false);
		connectionFactory = new ConnectionFactory();
		connectionPool = new GenericObjectPool<>(connectionFactory, poolConfig);
	}


	/**
	 * 查询初始化记录
	 */
	@GetMapping(value = "/test")
	public void test() {
		Connection connection = null;
		try {
			connection = connectionPool.borrowObject();
//			Thread.sleep((long) (Math.random() * 1000));
			Statement statement = connection.createStatement();
			ResultSet resultSet = statement.executeQuery("select * from a1_user");
			logger.info("查询mysql成功");
		} catch (Exception e) {
			e.printStackTrace();
			logger.error("线程启动出错");
		} finally {
			if (Objects.nonNull(connection)) {
				connectionPool.returnObject(connection);
			}
		}
	}

	/**
	 * 查询初始化记录
	 */
	@GetMapping(value = "/test2")
	public void test2() {
		Connection connection = null;
		try {
			connection = connectionFactory.create();
			Statement statement = connection.createStatement();
			ResultSet resultSet = statement.executeQuery("select * from a1_user");
			logger.info("查询mysql成功");
		} catch (Exception e) {
			e.printStackTrace();
			logger.error("线程启动出错");
		} finally {
			if (Objects.nonNull(connection)) {
				try {
					connection.close();
				} catch (SQLException e) {
					e.printStackTrace();
				}
			}
		}
	}
}
